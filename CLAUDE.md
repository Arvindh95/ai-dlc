# CLAUDE.md — Session bootstrap

This repo is an **AI-DLC source-of-truth requirements repo** (not a code repo). Built from AI-DLC Playbook v1.0.

## Mandatory first reads (in order)

Before doing **any** work in this repo, read these four files. Skipping this step wastes ~25k tokens grep-walking and risks wrong-folder actions.

1. `_index.md` — project metadata + folder map
2. `_prompts.md` — top-level routing rules (which folder for which request)
3. `<target-folder>/_index.md` — folder context, schema, catalog
4. `<target-folder>/_prompts.md` — valid operations in that folder

Total: ~6k tokens. Replaces ~30k tokens of blind exploration.

## Tier policy (3.2.0)

- **Tier A** (managed knowledge): `requirements/`, `spec/`, `design/`, `tasks/`, `feedback/`, `decisions/`, `incidents/`, `prompts/`, `deliverables/`, `dashboard/`, `raw-inputs/`, `signoffs/`, `change-requests/`, `00-overview/`, `adrs/`, repo root. **Both `_index.md` + `_prompts.md` mandatory.**
- **Tier B** (code/infra/scripts): `_index.md` mandatory, `_prompts.md` only if folder has real ops.
- **Tier C** (tool-managed, generated): exempt. See `SKIP_DIRS` in `scripts/regen_indexes.py`.

CI fails if any Tier A folder lacks either file.

## Globally forbidden

- Edit the `<!-- auto-managed below -->` region of any `_index.md` — it's regenerated by CI.
- Mutate record frontmatter (REQ/TASK/FB/etc.) without appending to its `## Change Log` section.
- Mark `status: approved` on a REQ without signoff evidence in `signoffs/`.
- Edit an approved REQ outside the change-request flow.
- Edit an accepted ADR — write a new ADR that supersedes it.
- Invent acceptance criteria, REQs, or other content not derivable from `raw-inputs/`.
- Commit without a work-item ID (REQ/TASK/FB/INC/CR/DEC) — the `.githooks/commit-msg` hook will reject.

## Operations live in `_prompts.md`, not here

Don't infer what to do from filenames. Match the user's request to a trigger phrase in the target folder's `_prompts.md`. If nothing matches, **escalate to the human — do not improvise.**

## CI gates

- `scripts/validate_frontmatter.py` — record schema validation
- `scripts/regen_indexes.py` — tier check + catalog regen (must be idempotent on commit)

Run both locally before committing:

```
python scripts/validate_frontmatter.py .
python scripts/regen_indexes.py .
git status   # should be clean
```
